# 몽중농원 🥕 (DreamFarm)

> *밀어낸 당근 한 조각에는, 누군가의 새벽이 들어 있었다.*

식탁 위 반찬을 말없이 밀어낸 아이가, 그날 밤 꿈속의 밭에서 눈을 뜬다.
**"네가 밀어낸 것을, 이번에는 네 손으로 길러 보아라."**
한 작물을 길러내는 과정을 통해 아버지의 노동과 사랑을 이해해 가는,
Pygame으로 만든 감성 내러티브 농장 게임입니다.

---

## 다운로드 (Windows)

[**Releases**](../../releases) 페이지에서 최신 `DreamFarm.exe`를 내려받아 바로 실행합니다.
별도 설치 없이 실행되는 단일 파일이며, 저장 파일은 exe와 같은 폴더에 생성됩니다.

- **Latest**로 표시된 릴리스가 정식 버전입니다.
- `-dev`가 붙은 릴리스는 **테스터용 프리릴리스**(최신 개발 빌드)입니다.

## 직접 실행 (개발)

```bash
pip install -r requirements.txt
python main.py
```

- Python 3.10+ / pygame(-ce) / numpy 필요
- 픽셀 폰트(Galmuri11)는 저장소에 포함되어 있으며, 없으면 자동 다운로드합니다.
- 오디오 장치가 없는 환경에서도 자동으로 무음 처리되어 정상 실행됩니다.

## 조작법

| 입력 | 동작 |
|------|------|
| 마우스 클릭/드래그 | 행동 선택, 손맛 조작(물주기·잡초·수확 등), 대화 진행 |
| 스페이스바 | 텍스트 진행 / 넘기기 (엔딩 크레딧은 꾹 눌러 게이지로 넘김) |
| **ESC** | 설정 열기 (메뉴 화면에서는 뒤로가기) |
| **M** | 음소거 토글 |
| **F11** | 전체 화면 토글 |
| **R** | (엔딩 화면) 다시하기 |
| 1~3 | (1회 클리어 후) 엔딩 화면에서 진/노멀/배드 감상 |

## 핵심 특징

- **읽고 판단하는 밭 관리** — 정답을 알려주지 않습니다. 수분·건강·잡초·해충·배수·스트레스
  여섯 스탯과 밭의 '증상'을 읽고 스스로 판단합니다.
- **기다림의 게임** — 작물은 제대로 된 돌봄과, 밭이 평온할 때의 '기다리기'에서만 자랍니다.
  쉬지 않고 5턴을 일하면 돌발 상황을 제한 시간 안에 수습해야 합니다.
- **태도로 갈리는 엔딩** — 점수가 아니라 인내·공감·이해가 결말(진·노멀·배드)을 정하고,
  끝내 밭을 잃으면 '시듦'으로 마무리됩니다. 결과 화면이 '왜 이 엔딩인지'를 알려줍니다.
- **여러 작물** — 아무 엔딩이나 한 번 보면 사과나무·감자·벼가 열립니다. 작물마다 밭의 성질과
  성장 그래픽, 미니게임(가지치기·굼벵이 잡기·모내기·거름 주기)과 수확 방식이 다릅니다.
- **업적** — 마인크래프트 도전과제 풍의 잠금 해제식 업적(수확·완벽 수확·전 작물·진엔딩 등).
  해제 시 토스트로 알리고 갤러리 "업적" 탭에서 모아 봅니다.
- **악)몽중농원** — 진엔딩을 보면 열리는 지옥 모드. 검붉은 밭과 확장 화면(960×720)에서
  남긴 음식을 다시 길러 거두어야 꿈에서 놓여납니다.
- **저장 / 갤러리** — 수동 저장·이어하기·자동 저장, 확인 절차, 그리고 엔딩·사건·회상·업적 갤러리.
- **절차적 사운드** — 모든 효과음·배경음을 numpy로 런타임 합성합니다(외부 음원 파일 0개).

## 코드 구조

```
main.py             게임 루프 · 씬 전환 · 씬별 배경음 매핑 · 오버레이/ESC 라우팅 · 좌표 스케일링
core/
  game_state.py     전역 상태, 엔딩 판정, 선택형 이벤트, 텍스트 데이터
  audio.py          numpy 절차적 오디오 엔진 (효과음·배경음 전부 런타임 합성)
  assets.py         픽셀 스프라이트 생성, 폰트, 작물 그래픽, 배경 렌더링
  save_system.py    세이브/메타(엔딩·클리어 횟수·업적) 저장
  achievements.py   업적 정의·해제·토스트
  settings_overlay.py  설정/저장 오버레이 (톱니 · ESC)
  ui.py             패널·버튼·미터 등 공용 UI 위젯
  version.py        버전 중앙 관리
scenes/             title · name_input · intro · crop_select · farm · stage1~4 · tending ·
                    story_choice · memory · epiphany · star_connect · father_day ·
                    gallery · ending · transition
```

## 배포 / 기여 메모

- 배포는 **git 태그 push**로 자동화됩니다(`.github/workflows/release.yml`). `-dev`/`-beta`/`-rc`
  태그는 프리릴리스, 그 외 `vX.Y.Z`는 정식 릴리스(Latest)로 올라갑니다.
- 브랜치: `dev`(개발) → `main`(안정). 버전은 `core/version.py`에서 관리합니다.
- 변경 내역은 [`CHANGELOG.md`](CHANGELOG.md)에, 개발 인수인계·노하우는 [`HANDOFF.md`](HANDOFF.md)에
  정리되어 있습니다.
