# UI/UX 디자인 일관성 감사 & 로드맵 (향후 계획)

몽중농원의 화면은 지금 **서로 다른 3가지 시각 언어가 섞여** 있어 통일감이 약하다. 이 문서는 무엇이
어긋나는지 정리하고, 하나의 스타일로 맞추기 위한 방향과 우선순위를 제안한다. (실제 리워크는 방향
확정 후 진행.)

## 1) 지금 섞여 있는 시각 언어

| 종류 | 렌더 방식 | 예시 |
|---|---|---|
| **픽셀 아트 스프라이트** | `create_sprite_from_string`(도트 → 사각형), scale 2~7 | 당근, 씨앗, 잡초, 돌, 새싹, 벌레, 아빠(폴백), 통·물뿌리개, mini 아이콘 |
| **벡터/부드러운 도형** | `pygame.draw` 원·타원·다각형(안티에일리어싱) | **사과·감자·쌀밥**, 날씨 아이콘, 톱니바퀴, 메달, 미터바, 둥근 패널·버튼, 배경 그라데이션 |
| **래스터 이미지** | PNG/JPG 로드·스케일 | `dad.png`(160²), `field_bed.jpg`, `logo.png` |
| **픽셀 폰트** | Galmuri11 11~44px | 전 화면 텍스트 |

## 2) 구체적으로 어긋나는 지점

1. **작물 4종의 스타일 불일치** — 당근만 **픽셀 스프라이트**, 사과·감자·쌀밥은 **부드러운 벡터**
   (매끈한 원/타원). 나란히 두면 당근만 도트, 나머지는 매끈해 이질감이 크다. (엔딩·수확·선택 화면 등
   작물이 크게 나오는 곳에서 특히 눈에 띔.) → **가장 먼저 손봐야 할 곳.**
2. **비정수 배율 스케일링으로 픽셀이 뭉개짐** — 800×600 가상 화면을 실제 해상도로 `pygame.transform.scale`
   (최근접)하지만, 4:3 레터박스 배율이 보통 2.4배·2.67배처럼 **정수가 아니라서** 도트 크기가 들쭉날쭉해
   진다("픽셀 느낌이 안 남"의 큰 원인). → **정수 배율**로 키우고 남는 여백만 레터박스하면 즉시 또렷해짐.
3. **매끈한 그라데이션 배경 vs 픽셀 스프라이트** — 하늘·잔디·흙을 연속 그라데이션으로 그려 도트와 안 어울림.
4. **둥근 모서리 패널·버튼(안티에일리어싱)** — 픽셀 UI라기보다 모던 카드에 가깝다.
5. **아이콘 제각각** — 날씨·톱니바퀴·메달·별은 벡터, 게임 오브젝트는 픽셀 → 아이콘 언어가 두 갈래.
6. **래스터 에셋** — `dad.png`·`field_bed.jpg`는 사진/일러스트풍이라 도트 톤과 결이 다르다.
7. **작은 스케일 픽셀** — mini 아이콘(scale 2)·13px 폰트는 도트가 너무 작아 픽셀감이 약하다.

## 3) 방향 옵션 (택1 — 정체성 결정)

- **A. 풀 픽셀 아트 통일** — 작물 벡터→픽셀, UI도 픽셀(각진 패널·도트 테두리·디더링 배경), 정수 배율.
  레트로 정체성 최강, 스팀 '픽셀 감성 농장' 태그와 잘 맞음. **작업량 최대.**
- **B. 클린/코지 통일(벡터·일러스트)** — 픽셀 스프라이트를 일관된 벡터/일러스트로 교체, 매끈한 방향 유지.
  픽셀 정체성은 포기. 모던·부드러운 인상.
- **C. 하이브리드 + 규칙 (권장)** — **게임 오브젝트=픽셀 아트, UI 크롬=클린**으로 역할을 나누되 **엄격한
  규칙**을 둔다: ①작물·아이콘 등 오브젝트는 전부 픽셀 스프라이트로 통일 ②정수 배율 최근접 스케일링
  ③팔레트 상수화(공용 팔레트에서만 색 선택) ④패널/버튼/폰트 스케일 규칙 일원화. 현재의 아늑한 매력을
  유지하면서 이질감만 제거 — **투자 대비 효과가 가장 좋음.**

## 4) 권장 로드맵 (방향 C 기준, 우선순위)  — 방향 **C 확정**

- **P1 — 즉효 개선  ✅ 완료 (dev.7)**
  - **작물 통일**: 사과·감자·쌀밥을 **픽셀 스프라이트**로 다시 그려 당근과 톤·질감을 맞춤
    (`core/assets.py` `food_apple`/`food_potato`/`food_rice`, `draw_crop_food`은 정수 배율로만 확대).
  - **정수 배율 스케일링(선별 적용)**: 2배 이상이 들어가는 고해상도(1440p·4K)에서만 정수 배율로 스냅해
    픽셀을 또렷하게(`game_main.py` `_apply_scaling`). 720p·1080p는 기존처럼 꽉 채워 **작아지는 리그레션 없음**.
- **P2 — 아이콘·색 일관화**
  - 날씨·톱니·메달·별 아이콘을 **하나의 스타일**(픽셀 권장)로 통일.
  - **팔레트 모듈화**: 흩어진 RGB 리터럴을 공용 팔레트 상수로 모아 색 통일(코드 그림도 팔레트 안에서만).
- **P3 — 래스터 에셋 정합**
  - `dad.png`·`field_bed.jpg`를 픽셀 아트로 교체하거나, 최소한 도트 톤에 맞게 처리(포스터라이즈·다운스케일).
- **P4 — UI 크롬 규칙화**
  - 패널·버튼의 모서리·테두리·그림자·여백·폰트 크기 규칙을 한 곳(디자인 토큰)으로 정의해 전 화면 적용.
  - 폰트 최소 크기 가이드(너무 작은 픽셀 폰트 지양), 배경 그라데이션은 도트 톤과 조화되게 밴딩/디더.

## 5) 참고 (관련 코드 위치)

- 픽셀 스프라이트: `core/assets.py` `create_sprite_from_string`, `init_sprites`
- 작물 그리기(불일치 핵심): `core/assets.py` `draw_crop_food`(당근=스프라이트, 그 외=`pygame.draw`)
- 배경/흙: `core/assets.py` `draw_tiled_background`
- 스케일링: `game_main.py` `_apply_scaling` / `pygame.transform.scale(...)`(정수 배율로 바꿀 지점)
- 패널·버튼·미터: `core/ui.py`
- 래스터: `core/dad.png`, `core/field_bed.jpg`, `core/logo.png`

> 방향(A/B/C)만 정해 주면 P1부터 순서대로 실제 리워크에 착수한다. 권장은 **C**(하이브리드+규칙): 현재
> 매력을 지키면서 가장 적은 리스크로 통일감을 확보한다.
