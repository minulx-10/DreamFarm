# Antigravity 에이전트 작업 지침

이 문서는 사용자의 요청에 따라 에이전트가 지속해서 지켜야 할 규칙을 명시한 문서입니다.

## 📌 규칙 사항
1. **계획서 및 보고서 한국어 작성**
   - 개발에 필요한 모든 계획서(`implementation_plan.md`), 작업 목록(`task.md`), 작업 완료 보고서(`walkthrough.md`) 등 모든 프로젝트 기획 및 작업 관리 문서는 **반드시 한국어로 작성**해야 합니다.
2. **코드 및 주석 지침**
   - 사용자 피드백을 반영하여 코드 개선 작업을 수행하고, 변경 후에는 정상 동작 여부를 테스트하여 원격 저장소(`main` 브랜치)에 반영합니다.
3. **핫픽스 및 마이너 패치 태깅 지침 (Git Tagging Rule)**
   - 이미 배포된 마이너 버전에 추가적인 핫픽스나 버그 수정을 반영하여 배포할 때는, 기존 태그(예: `v1.0.4`)를 덮어쓰지 말고 **하위 마이너 번호를 부여(예: `v1.0.4.1`, `v1.0.4.2` 등)하여 태그를 새로 생성하고 푸시**해야 합니다.
   - 이는 플레이어나 테스터가 구버전 빌드와 핫픽스 빌드를 혼동하지 않도록 돕고 배포 변경 추적성을 높이기 위한 규칙입니다.
